Glossary of OOP Terms

  1. Factory – Creational design pattern that allows an interface or class to create an object. Subclasses decide which class or object to instantiate.
  2. Instantiation – is the initial creation of the object
  3. Object – The self-contained component that contains methods and properties.
  4. Class – Blueprint for creating objects
  5. Namespace – The collection of defined symbolic names that include the information about the object that each name references
  6. Constructor – Code that prepares the new object each time the instance of a class gets created
  7. Fixture – Fixed state of a set of objects used as a baseline for running tests.
  8. Type hint – A solution to statically indicate the type of a value in Python code.
  9. Type cast – Specify the type of a variable
  10. Unit test – Testing the individual parts of the program or application.
  11. Static – Member is shared by all objects of the class.
  12. Static method – free to accept an arbitrary number of parameters besides self and cls parameters. Global method in a class.
  13. Class method – takes cls parameter that points to the class and not the object instance when the method is called.
  14. Facade – the exterior of the program
  15. Public – All classes and subclasses have access to the variable
  16. Private – The variable can’t be accessed by other classes and that one the class or subclass holding the variable can access it.
  17. Protected – Certain classes and subclasses have access to the variable
  18. Method – The actions or behaviors of the class defined in a function
  19. Property – Is a member that provides an adjustable method to read, write, or compute the value of a private field.
  20. Inheritance –mechanism where a class can derive from another class and the hierarchy of classes share a set of attributes and methods.
